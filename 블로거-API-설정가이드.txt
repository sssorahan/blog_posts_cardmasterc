 STEP 2. Google Cloud Console 설정
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. https://console.cloud.google.com 접속 및 로그인

2. 새 프로젝트 생성
   - 상단 프로젝트 선택 → [새 프로젝트]
   - 이름: "blogger-auto" (자유롭게 입력)
   - [만들기] 클릭

3. Blogger API 활성화
   - 좌측 메뉴 → [API 및 서비스] → [라이브러리]
   - 검색창에 "Blogger API v3" 입력
   - [Blogger API v3] 클릭 → [사용 설정]

4. OAuth 동의 화면 설정
   - [API 및 서비스] → [OAuth 동의 화면]
   - User Type: [외부] 선택 → [만들기]
   - 앱 이름: "Blogger Uploader" (자유롭게)
   - 사용자 지원 이메일: 본인 이메일 입력
   - 나머지는 기본값 → [저장 후 계속] 3번 클릭
   - [대시보드로 돌아가기]

5. OAuth 2.0 클라이언트 ID 만들기
   - [API 및 서비스] → [사용자 인증 정보]
   - [+ 사용자 인증 정보 만들기] → [OAuth 클라이언트 ID]
   - 애플리케이션 유형: [데스크톱 앱]
   - 이름: "blogger-uploader" (자유롭게)
   - [만들기]

6. client_secrets.json 다운로드
   - 생성된 클라이언트 ID 우측 ↓ (다운로드) 클릭
   - 파일명을 "client_secrets.json" 으로 변경
   - C:\Users\sora\blog-posts\ 폴더에 저장

 STEP 3. 블로그 ID 저장
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Blogger 대시보드 접속: https://www.blogger.com
2. 내 블로그 선택
3. 브라우저 URL 확인:
   blogger.com/blog/posts/[숫자] ← 이 숫자가 블로그 ID

4. C:\Users\sora\blog-posts\blog_id.txt 파일 생성
   파일 내용: 블로그 ID 숫자만 입력 (예: 1234567890123456789)

   Claude Code에서 생성 방법:
   "blog_id.txt에 내 블로그 ID [숫자] 를 저장해줘" 라고 요청

허용하면 blogger_token.json이 생성되고, 이후 자동 인증됩니다.

Q. "앱이 확인되지 않음" 경고가 떠요.
A. [고급] → [blogger-uploader(으)로 이동] 클릭하면 됩니다.
   본인이 만든 앱이므로 안전합니다.

Q. 업로드했는데 블로거에서 안 보여요.
A. Blogger 대시보드 → [게시물] → [예약됨] 탭을 확인하세요.
   미래 날짜로 예약된 포스트는 예약됨에 표시됩니다.

Q. 같은 파일을 실수로 두 번 실행하면요?
A. uploaded_posts.json에 이력이 기록되어 이미 업로드된 파일은
   자동으로 건너뜁니다.

Q. 인증 토큰이 만료됐어요.
A. blogger_token.json 파일을 삭제하고 다시 실행하면
   브라우저 인증이 다시 열립니다.
